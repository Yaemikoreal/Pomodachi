# 🐾 Pomodoro Pet 整体设计方案

## 1. 项目愿景与定位

*   **一句话定位：** 一款基于 Tauri 的本地优先、游戏化 AI 番茄钟桌面宠物。
*   **核心痛点：** 传统番茄钟太枯燥，容易中途放弃；自律靠意志力，缺乏正向反馈。
*   **解决方案：** 把专注时间具象化为宠物的生命值和成长。你专注，宠物就开心长大；你摸鱼，宠物就受伤生气。引入 AI 作为“宠物灵魂”，提供拟人化的情绪反馈和专注建议。
*   **目标用户：** 程序员、自由职业者、学生、注意力容易分散的效率工具控。

## 2. 技术架构设计

采用 **Tauri v2** 标准架构，前后端分离，本地优先。

```text
┌───────────────────────────────────────────────────────┐
│                    Tauri Runtime                       │
│                                                       │
│  ┌─────────────────┐        ┌──────────────────────┐ │
│  │   Frontend (UI)  │ IPC通信 │   Backend (Rust)     │ │
│  │   React / Vue    │◄──────►│   状态机 & 逻辑核心   │ │
│  │   - 宠物动画      │        │   - 番茄钟计时器      │ │
│  │   - 状态面板      │        │   - 窗口/进程监听     │ │
│  │   - 设置页        │        │   - 数据持久化        │ │
│  └─────────────────┘        └──────────────────────┘ │
│                                       │               │
│                                       ▼               │
│                             ┌──────────────────────┐ │
│                             │   AI Interface        │ │
│                             │   - Local LLM (Ollama)│ │
│                             │   - Cloud API         │ │
│                             └──────────────────────┘ │
└───────────────────────────────────────────────────────┘
```

### 技术栈选型
*   **桌面框架：** Tauri v2 (Rust)
*   **前端框架：** React + TypeScript + TailwindCSS + Framer Motion (丝滑动画)
*   **后端核心：** Rust
*   **本地存储：** SQLite (通过 `rusqlite` 或 Tauri 插件)
*   **AI 接入：** 兼容 OpenAI 格式 (支持本地 Ollama 或云端 API)
*   **进程监听：** `sysinfo` (Rust crate，用于检测前台应用)

## 3. 核心功能模块设计

### 3.1 桌面宠物系统
*   **展示形式：** 极小无边框透明窗口，始终置顶。
*   **点击穿透：** 宠物以外的区域鼠标事件穿透到桌面（Tauri API: `set_ignore_cursor_events`）。
*   **宠物状态机：**
    *   `Idle` (空闲)：溜达、打哈欠。
    *   `Focusing` (专注中)：戴眼镜看书、敲键盘，周围闪烁经验值特效。
    *   `Distraction` (分心)：着火、流泪、举牌抗议。
    *   `Resting` (休息)：吃东西、睡觉。
*   **成长系统：** 经验值 (EXP) + 等级 (Level) + 金币。专注获得 EXP，升级解锁新皮肤/房间背景。

### 3.2 番茄钟引擎
*   **核心逻辑：** 25分钟专注 + 5分钟短休息 + 15分钟长休息（可自定义）。
*   **状态流转：** 
    *   开启专注 -> 宠物进入 `Focusing`，启动分心监听守护线程。
    *   专注完成 -> 结算奖励 (EXP + 随机掉落宝物)，宠物进入 `Resting`。
    *   专注失败 -> 扣除宠物 HP，宠物进入 `Distraction`。

### 3.3 分心检测器 - *项目亮点*
这是 Rust 后端的核心价值所在，实现“你一摸鱼，宠物就受伤”。

*   **方案 A（MVP推荐）：进程名/窗口标题监控**
    *   Rust 后台线程每 3 秒调用系统 API，获取当前活动窗口标题或进程名 (如 `steam.exe`, `WeChat.exe`)。
    *   匹配用户自定义的“摸鱼黑名单”。命中则判定分心，触发惩罚事件。
*   **方案 B（高阶）：浏览器扩展通信**
    *   开发配套的 Chrome/Firefox 扩展，监听当前 Tab 的 URL。
    *   通过本地 WebSocket 将 URL 传给 Tauri 后端，精准识别 B站/微博等特定网页。

### 3.4 AI 灵魂注入 - *贴合热点*
将 AI 作为宠物的“大脑”，而不是生硬的对话框。

*   **情绪化反馈：** 当检测到分心或专注完成时，调用 LLM 生成宠物的吐槽或鼓励。
    *   *Prompt 示例:* "你是一个傲娇的像素小猫，主人刚才专注了25分钟写代码，请用傲娇的语气夸奖他，不超过30字。"
*   **每日专注报告：** 每天下班时，AI 汇总当天的番茄数据，生成趣味战报。
    *   *示例:* "今天你摸鱼3次，主要集中在下午3点，主要是被B站吸引。建议明天下午3点喝杯咖啡抗住！"
*   **智能伴侣：** 点击宠物弹出小聊天框，可以和宠物闲聊，宠物知道你今天的工作状态。

## 4. 数据模型设计

使用 SQLite 进行本地持久化。

```sql
-- 宠物状态表 (只有一条记录)
CREATE TABLE pet (
    id INTEGER PRIMARY KEY DEFAULT 1,
    name TEXT DEFAULT 'Pixel',
    level INTEGER DEFAULT 1,
    exp INTEGER DEFAULT 0,
    hp INTEGER DEFAULT 100,
    mood TEXT DEFAULT 'idle', -- idle, happy, angry
    skin_id TEXT DEFAULT 'default_cat'
);

-- 专注记录表
CREATE TABLE pomodoro_record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    start_time DATETIME,
    end_time DATETIME,
    duration_min INTEGER,
    status TEXT, -- completed, failed
    fail_reason TEXT -- distraction, manual_cancel
);

-- 分心黑名单表
CREATE TABLE blocklist (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    app_name TEXT, -- e.g., steam.exe
    website_url TEXT -- e.g., youtube.com
);

-- AI 对话历史表
CREATE TABLE chat_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    role TEXT, -- user, assistant
    content TEXT,
    created_at DATETIME
);
```

## 5. UI/UX 交互设计

### 5.1 视图拆分
1.  **Pet Window (悬浮窗 - 常驻):** 150x150 像素，透明背景，只显示宠物动画和极简状态（如血条、倒计时）。
2.  **Dashboard Panel (侧边栏 - 按需展开):** 点击宠物时滑出，展示详细数据、设置项和 AI 聊天框。
3.  **System Tray (托盘区):** 右键菜单（开始专注、暂停、退出、显示/隐藏）。

### 5.2 交互流设计
*   **开始专注：** 右键托盘 -> Start，或展开面板点击“开始”。宠物带上护目镜，倒计时浮现。
*   **摸鱼被抓：** 用户切换到游戏 -> Rust 后台检测到 -> 通知前端 -> 宠物动画变为“暴怒/哭泣” -> 血条扣减 -> 弹出气泡提示："再玩我就离家出走！"
*   **完成专注：** 播放金币掉落音效 -> 经验条上涨 -> 宠物欢呼。

## 6. 开发路线图

### Phase 1: 骨架与呼吸 (Week 1-2) - MVP
*   初始化 Tauri v2 + React 项目。
*   实现无边框、置顶、透明的悬浮窗，跑通点击穿透。
*   引入像素小猫 Sprite 动画 (Idle & Focusing)。
*   实现基础番茄钟计时逻辑（前端驱动即可）。

### Phase 2: 赋予生命 (Week 3-4) - 游戏化
*   将计时逻辑移至 Rust 后端，防止前端休眠被杀。
*   实现 SQLite 数据存储，构建经验/等级系统。
*   开发 `sysinfo` 进程监听模块，实现黑名单匹配。
*   联动：进程监听触发宠物状态改变 (Distraction) 和 HP 扣减。

### Phase 3: 注入灵魂 (Week 5-6) - AI 赋能
*   开发侧边栏面板。
*   接入 OpenAI 兼容 API (支持配置 Base URL 和 Key，适配 Ollama)。
*   实现宠物的情绪反馈（被抓包时的吐槽、完成时的夸奖）。
*   实现简易 Chat 界面，可与宠物对话。

### Phase 4: 打磨与出海 (Week 7+)
*   系统托盘交互优化。
*   多宠物/皮肤支持（用金币购买）。
*   编写 README 中的动图展示、打包发布 v1.0。
