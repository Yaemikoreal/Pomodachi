# ADR 0002: 设置面板与任务面板第一阶段功能范围

## 状态

已接受

## 背景

用户反馈当前设置面板仅有 LLM 相关的 API Base、Model、API Key 三项，任务面板仅有列出/完成/延后/删除四项，功能过于简陋，需要明确第一阶段应补齐哪些功能点。

## 决策

### 设置面板第一阶段范围

按功能分组，第一阶段实现以下三组：

**A. LLM 伙伴**
- Provider 类型（OpenAI / 自定义 OpenAI 兼容）
- API Base
- Model
- API Key
- 系统提示词（允许用户微调角色人设）

**B. 角色形象**
- 资源包选择 / 资源路径
- 整体缩放比例（`0.5x ~ 1.5x`，默认 `1.0x`）
- 边缘修复开关（PMA/bleed 兜底）

**C. 窗口生命周期**
- 开机自启
- 总是置顶
- 显示/隐藏快捷键
- 点击穿透总开关

第二阶段再考虑通知与音效、数据与隐私相关选项。

### 任务面板第一阶段范围

按功能分组，第一阶段实现以下三组：

**A. 任务查看**
- 按截止时间排序
- 显示情绪提示
- 显示重复规则

**B. 任务操作**
- 完成
- 删除
- 延后（支持自定义分钟数，不只 5 分钟）
- 编辑标题、截止时间、重复规则、情绪提示
- 手动添加任务

**C. 过滤与视图**
- 全部 / 未完成 / 已完成
- 已过期任务高亮

第二阶段再考虑 LLM 创建来源标记、批量操作等。

## 后果

### 正面

- 第一阶段聚焦核心功能，避免面板过度复杂；
- 与 ADR 0001 中的尺寸、渲染修复、点击穿透开关形成完整闭环；
- 后端已有 `create_task`、`complete_task` 等命令，前端补齐表单和编辑即可实现大部分操作。

### 负面

- 用户无法在第一阶段调整通知音效、隐私清理等选项；
- 任务面板暂不支持批量操作和按标签分类。
