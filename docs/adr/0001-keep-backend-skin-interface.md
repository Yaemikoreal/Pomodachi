# ADR-0001：保留后端皮肤接口

## 状态

已接受

## 背景

test_pet 分支将宠物从 Spine 骨骼动画替换为 18x18 像素宠物。原系统有完整的皮肤切换功能：
- 后端：`pet.rs` 的 `set_pet_skin` / `get_pet_skin` 方法
- IPC：`lib.rs` 的 `set_pet_skin` / `get_pet_skin` 命令
- 数据库：`pet` 表的 `skin_id` 字段
- 前端：`Sidebar.tsx` 的皮肤选择器

需要决定如何处理这些代码。

## 决策

**保留后端接口，禁用前端 UI**

- 保留 `pet.rs` 的 skin 方法（不删除）
- 保留 `lib.rs` 的 IPC 命令（不删除）
- 保留数据库 `skin_id` 字段（不删除）
- 在 `Sidebar.tsx` 中隐藏/禁用皮肤选择器 UI
- `PixelPet` 组件不接受 `skinId` prop

## 原因

1. **向后兼容性**：删除后端接口会破坏现有数据和可能的外部调用
2. **降低风险**：test_pet 分支专注前端验证，后端保持稳定
3. **可恢复性**：将来如果想恢复皮肤系统，不需要重新编写后端代码
4. **最小改动原则**：只改必要的部分，减少引入 bug 的风险

## 后果

### 正面
- 后端代码稳定，测试不受影响
- 数据库迁移简单（无需删除字段）
- 将来恢复皮肤系统成本低

### 负面
- 保留未使用的代码（轻微技术债务）
- 新开发者可能困惑：为什么有接口但没有 UI
